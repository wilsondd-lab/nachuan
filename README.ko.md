# 纳川.skill

> To be anyone, to be better you.

[中文](README.md) · [English](README.en.md) · [日本語](README.ja.md) · [Español](README.es.md) · [Français](README.fr.md) · [Deutsch](README.de.md) · [Português](README.pt.md) · [العربية](README.ar.md)

**Distill Everything.skill** 은 **纳川.skill** 의 영어 이름입니다.

纳川.skill 은 Claude Code 를 위한 범용 자료 증류 Skill 입니다. PDF, 슬라이드, 웹페이지, 회의 기록, 리서치 자료, 초안, 흩어진 메모를 먼저 읽고, 재사용 가능한 지식, 구조, 콘텐츠 자산으로 정리합니다.

프롬프트를 여러 개 쌓아 둔 도구가 아닙니다. 먼저 스캔하고, 다음 처리 경로를 고른 뒤, 결과를 다시 쓸 수 있는 형태로 남기는 워크플로입니다.

---

## 이름의 의미

纳川 은 여러 흐름을 받아들인다는 뜻을 담고 있습니다.

실제 업무 자료는 깔끔하게 정리되어 들어오지 않습니다. 문서, 도표, 링크, 전사문, 아이디어 조각이 섞여 있습니다. 纳川.skill 은 바로 요약하지 않고, 먼저 자료 안에 무엇이 있는지와 어떤 경로가 가장 유용한지 판단합니다.

**To be anyone, to be better you** 는 다른 사람을 복사한다는 뜻이 아닙니다. 다양한 자료에서 방법, 구조, 판단, 표현 방식을 흡수해 자신의 일을 더 잘하게 만든다는 뜻입니다.

---

## 세 가지 강점

| 강점 | 해결하는 문제 | 사용자가 느끼는 점 |
|---|---|---|
| **질문 전에 스캔** | 자료가 복잡하면 무엇을 요청해야 할지 모를 수 있음 | 자료 개요와 2-4 개의 처리 선택지를 먼저 제시 |
| **네 가지 파이프라인** | 요약, 구조화, 지식화, 생성이 뒤섞이기 쉬움 | 목적과 산출물이 분명한 경로로 처리 |
| **결과를 자산으로 저장** | AI 출력이 한 번의 대화로 끝나기 쉬움 | Markdown, 태그, 링크, 템플릿으로 재사용 가능 |

---

## 네 가지 파이프라인

| Path | 이름 | 사용할 때 | 대표 산출물 |
|---|---|---|---|
| **A** | Essence Distillation | 핵심 주장, 판단, 재사용 가능한 원칙이 필요할 때 | 요약, 판단 목록, 원칙 카드 |
| **B** | Structure Decomposition | 서사, 페이지, 논리 구조를 파악하고 싶을 때 | 구조 맵, 표현 패턴, 프레임워크 |
| **C** | Knowledge Extraction | 자료를 장기 지식 베이스로 만들고 싶을 때 | Markdown 노트, YAML 태그, `[[wikilinks]]` |
| **D** | Content Generation | 추출한 지식으로 새 표현을 만들고 싶을 때 | 카피, 스크립트, 아웃라인, 설명문, 템플릿, 초안 |

---

## 워크플로

1. **자료 투입**: 문서, 링크, 전사문, 메모, 혼합 자료.
2. **조용히 스캔**: 유형, 규모, 구조, 키워드, 잠재 가치를 파악.
3. **경로 선택**: 추천 경로와 대안을 제시.
4. **파이프라인 실행**: A/B/C/D 를 단독, 병렬, 또는 순차로 실행.
5. **결과 저장**: 검색, 링크, 재사용이 가능한 Markdown 으로 남김.

## 기원

纳川.skill 은 Huashu 의 Nuwa Skill 에서 영감을 받았습니다. Skill 은 프롬프트 그 이상이 될 수 있습니다. 경험, 판단, 경로 선택, 워크플로를 재사용 가능한 시스템으로 만들 수 있습니다.

그 영감을 바탕으로 실제 업무 장면에서 네다섯 차례 반복하며, 현장의 자료 처리 요구에 더 가까운 형태로 다듬었습니다.
