# 纳川.skill

> To be anyone, to be better you.

[中文](README.md) · [English](README.en.md) · [한국어](README.ko.md) · [Español](README.es.md) · [Français](README.fr.md) · [Deutsch](README.de.md) · [Português](README.pt.md) · [العربية](README.ar.md)

**Distill Everything.skill** は **纳川.skill** の英語名です。

纳川.skill は Claude Code のための汎用的な素材蒸留 Skill です。PDF、スライド、Web ページ、文字起こし、調査資料、下書き、断片的なメモを読み取り、再利用できる知識、構造、コンテンツ資産へ変換します。

単なるプロンプト集ではありません。まず素材を読み、次に処理ルートを選び、最後に成果を再利用できる形で残すワークフローです。

---

## 名前に込めた意味

纳川 は、多くの流れを受け入れるという意味を持ちます。

実際の仕事では、入力はきれいに整理されていません。文書、図表、リンク、転写、断片的な考えが混ざっています。纳川.skill はすぐに要約せず、まず素材の中身とつながりを見て、どの処理が最も有効かを判断します。

**To be anyone, to be better you** は誰かをコピーすることではありません。多様な素材から方法、構造、判断、表現を吸収し、自分自身の仕事をより良くするという意味です。

---

## 3 つの特徴

| 特徴 | 解決すること | 体験 |
|---|---|---|
| **質問の前にスキャン** | 素材が複雑なとき、何を頼めばよいか分からない | 概要を示した上で、2-4 個の処理ルートを提示 |
| **4 つの処理ルート** | 要約、構造化、知識化、生成が混ざりやすい | 目的と成果物が明確なルートで処理 |
| **成果を資産化** | AI の出力が一回の会話で終わりがち | Markdown、タグ、リンク、テンプレートとして残せる |

---

## 4 つのパイプライン

| Path | 名称 | 使う場面 | 主な成果物 |
|---|---|---|---|
| **A** | Essence Distillation | 核心、判断、再利用できる原則を掴みたい | 要約、判断リスト、原則カード |
| **B** | Structure Decomposition | 叙述、ページ、議論の構造を理解したい | 構造マップ、表現パターン、フレームワーク |
| **C** | Knowledge Extraction | 素材を長期的な知識ベースにしたい | Markdown ノート、YAML タグ、`[[wikilinks]]` |
| **D** | Content Generation | 抽出した知識から新しい表現を作りたい | 文章、台本、アウトライン、説明、テンプレート、下書き |

---

## ワークフロー

1. **素材を入れる**：文書、リンク、転写、メモ、混合資料。
2. **静かにスキャンする**：種類、規模、構造、キーワード、価値を把握。
3. **ルートを選ぶ**：推奨ルートと代替案を提示。
4. **パイプラインを実行する**：A/B/C/D を単独、並行、または連続で実行。
5. **成果を残す**：検索、リンク、再利用ができる Markdown にする。

## 起源

纳川.skill は花叔の女娲 Skill に啓発されました。Skill はプロンプト以上のものになり得ます。経験、判断、ルート選択、ワークフローを再利用可能なシステムにできます。

その着想をもとに、実際の仕事の場面で 4-5 回の実践的な反復を行い、より現場の素材処理に近い形へ磨き込みました。
