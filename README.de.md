# 纳川.skill

> To be anyone, to be better you.

[中文](README.md) · [English](README.en.md) · [日本語](README.ja.md) · [한국어](README.ko.md) · [Español](README.es.md) · [Français](README.fr.md) · [Português](README.pt.md) · [العربية](README.ar.md)

**Distill Everything.skill** ist der englische Name von **纳川.skill**.

纳川.skill ist ein universeller Claude Code Skill zur Destillation von Material. Er liest PDFs, Präsentationen, Webseiten, Transkripte, Recherchepakete, Entwürfe und verstreute Notizen und verwandelt sie in wiederverwendbares Wissen, Struktur und Content-Assets.

Es ist keine Sammlung von Prompts. Es ist ein Workflow: zuerst scannen, dann routen, die passende Pipeline ausführen und das Ergebnis wiederverwendbar aufbewahren.

---

## Warum Nachuan

Nachuan steht für das Aufnehmen vieler Ströme.

Reale Arbeit kommt selten als saubere Eingabe. Sie besteht aus Dokumenten, Grafiken, Links, Transkripten, halben Ideen und Fragmenten. 纳川.skill versteht zuerst, was vorhanden ist, und entscheidet dann, welcher Weg das Material nützlich macht.

**To be anyone, to be better you** bedeutet nicht, jemand anderen zu kopieren. Es bedeutet, Methoden, Strukturen, Urteile und Ausdrucksweisen aus vielen Quellen aufzunehmen, damit die eigene Arbeit stärker wird.

---

## Drei Stärken

| Stärke | Gelöstes Problem | Erlebnis |
|---|---|---|
| **Scannen vor dem Fragen** | Bei chaotischem Material weiß man oft nicht, was man fragen soll | Es gibt zuerst Überblick und dann 2-4 konkrete Optionen |
| **Vier geroutete Pipelines** | Zusammenfassung, Struktur, Wissen und Generierung werden oft vermischt | Jede Route hat Ziel, Ergebnis und nächsten Schritt |
| **Output wird Asset** | Viele KI-Ergebnisse verschwinden in einem Chat | Markdown, Tags, Links und Vorlagen bleiben wiederverwendbar |

---

## Vier Pipelines

| Path | Name | Wann verwenden | Typisches Ergebnis |
|---|---|---|---|
| **A** | Essence Distillation | Für Kernaussagen, Urteile und wiederverwendbare Prinzipien | Zusammenfassung, Entscheidungsliste, Prinzipkarten |
| **B** | Structure Decomposition | Für Narrativ, Seite, Argumentation oder Aufbau | Strukturkarte, Ausdrucksmuster, Framework |
| **C** | Knowledge Extraction | Für eine langfristige Wissensbasis | Markdown-Notizen, YAML-Tags, `[[wikilinks]]` |
| **D** | Content Generation | Für neue Ausdrucksformen aus extrahiertem Wissen | Text, Skripte, Gliederungen, Erklärungen, Vorlagen, Entwürfe |

---

## Workflow

1. **Material hinzufügen**: Dokumente, Links, Transkripte, Notizen oder gemischte Pakete.
2. **Stilles Scannen**: Typ, Umfang, Struktur, Schlüsselwörter und Wert erkennen.
3. **Route wählen**: Empfehlung plus Alternativen erhalten.
4. **Pipeline ausführen**: A/B/C/D einzeln, parallel oder nacheinander nutzen.
5. **Ergebnis behalten**: als suchbares, verlinkbares und wiederverwendbares Markdown speichern.

## Ursprung

纳川.skill wurde von Huashus Nuwa Skill inspiriert. Die zentrale Idee: Ein Skill kann mehr sein als ein Prompt. Er kann Erfahrung, Urteil, Routenauswahl und Workflow in ein wiederverwendbares System verwandeln.

Auf dieser Inspiration aufbauend wurde 纳川.skill vier oder fünf praktische Iterationen lang verfeinert und näher an reale Anforderungen der Wissensarbeit gebracht.
