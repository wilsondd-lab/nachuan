# 纳川 · Markdown 知识库设置指南

## 为什么用 Markdown 知识库

纳川默认把长期知识沉淀到 Obsidian vault 或任意 Markdown 文件夹中：

- **纯文本**：未来可读，不绑定平台
- **wikilinks**：用 `[[笔记名]]` 显式连接知识
- **YAML tags**：方便按路径、主题、来源检索
- **本地优先**：材料和知识在自己的磁盘上
- **Git 友好**：可以做版本控制和备份

---

## 一、推荐目录

```text
your-vault/
└── 纳川/
    ├── 纳川_MOC.md
    ├── 学习报告/
    ├── 知识单元/
    ├── 精华萃取/
    ├── 结构拆解/
    ├── 生成内容/
    └── 路由日志/
```

新建专用目录：

```bash
mkdir -p ~/Documents/纳川-vault/纳川/{学习报告,知识单元,精华萃取,结构拆解,生成内容,路由日志}
```

## 二、笔记格式

### 知识单元

```markdown
---
tags: [纳川, 知识单元, Path-C, 主题标签]
created: YYYY-MM-DD
source: "[[来源学习报告]]"
related:
  - "[[相关笔记]]"
---

# 知识点标题

2-5 句记录事实、机制、判断或方法。

**来源**：第 N 页 / 第 N 段
**可复用场景**：说明未来可用于什么任务
```

### 学习报告

```markdown
---
tags: [纳川, 学习报告, Path-C, 主题标签]
created: YYYY-MM-DD
knowledge_count: N
topics: [主题1, 主题2]
---

# 材料名 · 学习报告

## 基本信息
- 类型：
- 规模：
- 主题：

## 知识提取成果
- [[知识单元1]]
- [[知识单元2]]

## 关键学习点
1. ...
2. ...
```

### 精华萃取

```markdown
---
tags: [纳川, 精华萃取, Path-A, 主题标签]
created: YYYY-MM-DD
source: "[[来源材料]]"
---

# 材料名 · 精华

## 核心观点
1. ...

## 关键判断
1. ...

## 可复用规律
1. ...
```

### 结构拆解

```markdown
---
tags: [纳川, 结构拆解, Path-B, 主题标签]
created: YYYY-MM-DD
source: "[[来源材料]]"
---

## 整体结构

## 关键模块

## 可复用模式
```

## 三、标签建议

每条笔记至少包含：

- `纳川`
- `Path-A` / `Path-B` / `Path-C` / `Path-D`
- 内容类型：`精华萃取` / `结构拆解` / `知识单元` / `内容生成`
- 主题标签：从材料中提取，不预设行业

---

## 四、预搜

开始处理前，搜索已有内容：

```bash
grep -rl "关键词" ~/Documents/纳川-vault/纳川/
```

也可以在 Obsidian 中使用全局搜索。

---

## 五、MOC 索引

`纳川_MOC.md` 可以维护一个总索引：

```markdown
# 纳川 · 知识索引

## 最近处理
- [[材料A_学习报告]]
- [[材料B_结构拆解]]

## 主题
- [[主题1]]
- [[主题2]]

## 可复用模板
- [[模板A]]
- [[模板B]]
```

---

## 六、写入确认

写入知识库前，纳川必须展示预览：

```text
准备写入：
- 学习报告：1 份
- 知识单元：N 条
- 精华摘要：N 条
- 结构模式：N 条

确认写入？【全部写入 / 仅知识 / 仅结构 / 暂不写入】
```
