# Path C · 知识提取协议（Obsidian 版）

## 定位

将任意文档（PPT/PDF/报告/竞品/会议记录/数据）中的**知识点**逐页/逐段提取，以 Markdown 笔记存入 Obsidian vault，通过 wikilinks 建立与已有知识的关联。

---

## 适用场景

| 输入 | 典型信号 |
|------|----------|
| PPT/PPTX | 产说会、产品发布、培训课件 |
| PDF | 白皮书、研报、政策文件、案例集 |
| 竞品文档 | 含"竞品""竞争对手"信号 |
| 会议记录 | 多人对话、决策纪要 |
| 数据/表格 | .csv/.xlsx，数字为主 |
| 纯文本/网页 | 长文、报告、政策解读 |

---

## 双层输出结构

### 层级 1：学习报告笔记

一份结构化 Markdown 笔记，存入 `vault/纳川/学习报告/`：

```markdown
---
tags: [纳川, 学习报告, 领域标签]
created: YYYY-MM-DD
knowledge_count: N
---

# 文档名 · 学习报告

## 基本信息
- **文档类型**：PPT / PDF / ...
- **规模**：N 页 / N 字

## 知识提取成果
### 主题 1（N 条笔记）
- [[笔记1标题]]
- [[笔记2标题]]

## 知识演进
| 旧笔记 | 新笔记 | 关系 |
|--------|--------|------|
| [[旧笔记]] | [[新笔记]] | enriches |

## 关键学习点
1. ...
2. ...

## 知识完整性
| 维度 | 覆盖 |
|------|:----:|
| 维度 1 | ✅ |
```

### 层级 2：独立知识笔记

每条知识一个独立 `.md` 文件，存入 `vault/纳川/知识记忆/[领域]/`：

```markdown
---
tags: [纳川, 领域标签, 子主题, Path-C]
created: YYYY-MM-DD
source: "[[来源学习报告]]"
related:
  - "[[相关笔记1]]"
  - "[[相关笔记2]]"
---

# 笔记标题：一句话概括知识点

[2-5 句展开：关键事实、数据、机制、背景]

**来源**：第 N 页
**可复用场景**：[在什么情况下有用]
```

## 完整流水线

### Phase 1：Vault 预搜

在开始提取前，搜索 vault 中已有知识：

```bash
# 按关键词搜索
grep -rl "关键词1" ~/Documents/纳川-vault/纳川/
grep -rl "关键词2" ~/Documents/纳川-vault/纳川/

# 在 Obsidian 中：Ctrl/Cmd+Shift+F 全局搜索
# 或在图谱视图查看相关笔记群
```

目的：避免创建重复笔记，发现可 wikilink 关联的已有笔记。

### Phase 2：逐页提取

逐页/逐段阅读文档，记录：

| 页码 | 主题 | 知识点 | 是否新增 | 建议标签 | 关联已有笔记 |
|------|------|--------|----------|----------|-------------|

### Phase 3：去重与演进判断

对每个候选知识点，与 Phase 1 预搜结果对比：

| 关系 | 判断标准 | 处理方式 |
|------|----------|----------|
| **全新** | vault 中无相关内容 | 创建新笔记 |
| **丰富（enriches）** | 比已有笔记更详细/有新的案例或数据 | 创建新笔记 + wikilink `[[已有笔记]]` |
| **演进（evolves）** | 推翻或升级已有认知 | 创建新笔记 + wikilink + 正文注明演进 |
| **重复** | 内容已被已有笔记覆盖 | 跳过 |
| **冲突** | 与已有笔记矛盾 | 创建笔记 + 标注 tension |

### Phase 4：构建 Wikilinks

在每个新笔记的 frontmatter `related` 字段中添加 wikilinks：

```yaml
related:
  - "[[已有笔记1]]"  # enriches
  - "[[已有笔记2]]"  # evolves
```

同时在已有笔记中反向添加链接（可选，保持双向链接完整性）。

### Phase 5：打标签

每条笔记的 YAML frontmatter 必须带标签：

```yaml
tags: [纳川, 领域标签, 子主题, Path-C, 内容类型]
```

标签层级：
1. `纳川` — 系统标识（必带）
2. 领域标签：`保险` / `养老` / `护理` / ...
3. 子主题：`产品设计` / `政策` / `销售` / ...
4. `Path-C` — 来源路径
5. 内容类型：`知识记忆` / `数据` / `案例` / `方法论` / ...

### Phase 6：写入（用户确认后）

展示预览 → 用户确认 → 创建笔记文件。

---

## 笔记质量标准

好的知识笔记：
- ✅ 有具象的数据/案例/金句
- ✅ 标注了来源页（可追溯）
- ✅ 说明了可复用场景
- ✅ 有 wikilinks 关联（不孤立）
- ✅ 文件命名清晰可搜索

不好的知识笔记：
- ❌ "这份文档讲的是养老产业" — 太泛
- ❌ 没有 wikilinks — 孤岛笔记
- ❌ 没有标签 — 无法被搜索
