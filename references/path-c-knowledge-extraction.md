# Path C · 知识提取

## 定位

Path C 将任意材料中的事实、机制、方法、判断和案例提取为 Markdown 知识单元，并通过 wikilinks 与已有知识建立关联。

---

## 适用输入

| 输入 | 典型内容 |
|------|----------|
| 文档 / PDF | 研究、说明、报告、案例 |
| PPT / deck | 页面信息、叙事节点、观点证据 |
| 网页 / 长文 | 观点、案例、方法、参考资料 |
| 转录 / 会议记录 | 决策、问题、共识、行动项 |
| 表格 / 数据 | 指标、变化、洞察、异常 |
| 多文件材料 | 共同主题、差异、互补信息 |

---

## 双层输出

### 1. 学习报告

存入 `vault/纳川/学习报告/`：

```markdown
---
tags: [纳川, 学习报告, Path-C, 主题标签]
created: YYYY-MM-DD
knowledge_count: N
topics: [主题1, 主题2]
---

# 材料名 · 学习报告

## 基本信息
- 类型：
- 规模：
- 主题：

## 知识提取成果
- [[知识单元1]]
- [[知识单元2]]

## 关键学习点
1. ...
2. ...
```

### 2. 独立知识单元

存入 `vault/纳川/知识单元/[主题]/`：

```markdown
---
tags: [纳川, 知识单元, Path-C, 主题标签]
created: YYYY-MM-DD
source: "[[来源学习报告]]"
related:
  - "[[相关笔记]]"
---

# 知识点标题

2-5 句记录事实、机制、判断或方法。

**来源**：第 N 页 / 第 N 段
**可复用场景**：说明这个知识未来可用于什么任务
```

## 流程

### Phase 1：预搜

先搜索知识库中的已有内容：

```bash
grep -rl "关键词" ~/Documents/纳川-vault/纳川/
```

目的：避免重复笔记，找到可关联的旧知识。

### Phase 2：候选知识点

逐页或逐段记录：

| 位置 | 主题 | 知识点 | 类型 | 建议标签 | 关联笔记 |
|------|------|--------|------|----------|----------|

### Phase 3：去重与关系判断

| 关系 | 判断标准 | 处理方式 |
|------|----------|----------|
| 全新 | 知识库中无相关内容 | 创建新笔记 |
| 丰富 | 比已有笔记更具体 | 新建并链接旧笔记 |
| 演进 | 更新或修正已有认知 | 新建并标注演进关系 |
| 重复 | 已被覆盖 | 跳过 |
| 冲突 | 与已有笔记矛盾 | 新建并标注 tension |

### Phase 4：写入前确认

展示准备写入的学习报告、知识单元、标签和 wikilinks，用户确认后再写入。

---

## 质量标准

好的知识单元：

- 有明确标题
- 有来源位置
- 有可复用场景
- 有标签
- 与已有知识建立链接

不好的知识单元：

- 只是泛泛摘要
- 没有来源
- 无法判断未来怎么复用
- 没有标签和链接
