# Path B · 结构拆解

## 定位

Path B 负责回答一个问题：这份内容是怎么组织起来的？

它不关心材料来自哪个行业，而是拆解通用结构：叙事顺序、信息层级、证据安排、节奏控制、页面或段落模式。

---

## 拆解维度

| 维度 | 核心问题 | 产出 |
|------|----------|------|
| 信息层级 | 先讲什么，后讲什么？ | 结构树、目录关系 |
| 叙事路径 | 观点如何推进？ | 叙事链、转折点 |
| 证据组织 | 如何支撑判断？ | 证据类型、案例位置 |
| 表达模式 | 哪些段落/页面套路可复用？ | 模式清单、模板句式 |
| 节奏控制 | 信息密度如何变化？ | 节奏图、轻重分布 |

---

## 常见结构模式

| 模式 | 说明 |
|------|------|
| 问题 → 原因 → 方案 | 先建立问题，再给解决路径 |
| 现状 → 洞察 → 行动 | 从事实进入判断，再落到动作 |
| 观点 → 证据 → 反证 | 强观点表达常用结构 |
| 案例 → 规律 → 复用 | 从个案提炼方法 |
| 对比 → 差异 → 选择 | 用对照帮助决策 |
| 背景 → 机制 → 影响 | 解释复杂系统或趋势 |

---

## 输出结构

1. **整体结构图**：材料从开头到结尾如何推进。
2. **关键模块拆解**：每个部分承担什么功能。
3. **表达模式清单**：可复用的标题、段落、页面或论证方式。
4. **可迁移框架**：把本材料的方法抽象成可复用模板。

---

## 写入合约
