# 纳川 · 访谈模板

## 设计原则

1. **先扫再问**：静默扫描内容 → 抓关键信号 → 匹配模板 → 给倾向性建议
2. **选择题，不开放题**：每个选项是一个完整的路径组合（如 "B+C 并行"），用户只需选
3. **推荐项在最前**：第一个选项 = 纳川判断最可能符合用户意图的路径
4. **总有"自定义"**：每个模板第 D 项永远是 ⚪ 自定义
5. **每条选项一句话说清理由**：不让用户猜每条是什么意思

---

## 信号扫描

在匹配模板前，先做静默扫描：

| 扫描维度 | 具体内容 |
|----------|----------|
| 文件类型 | .pptx / .pdf / .md / .txt / URL / .csv / .xlsx / 多文件混合 |
| 规模 | 页数 / 字数 / 文件数 |
| 结构特征 | PPT 有讲稿备注？/ 转录有明确说话人？/ 文档有章节？ |
| 关键词 | 产品名 / 人名 / "竞品" / "培训" / "产说会" / "蒸馏" / "融合" / "对比" / 领域词 |
| 领域 | 保险 / 养老 / 护理 / 投资 / 培训 / 通用 / 未知 |
| vault 预搜 | 搜到 [N] 条相关笔记？有哪些主题？ |

## 模板匹配逻辑

```
if 用户说"蒸馏 XX" or "融合 XX":
    → 蒸馏专属模板（情况一）
elif 用户说"蒸馏 XX" + 提供了内容资料:
    → 蒸馏专属模板（情况二）
elif 文件是 .pptx:
    if 有讲稿备注 + 可识别演讲者:
        → 模板 2
    else:
        → 模板 1
elif 文件是 .txt/.md 转录/演讲:
    if 单一明确说话人:
        → 模板 3
    elif 多人对话:
        → 模板 10
    else:
        → 模板 9（散装笔记）
elif 用户提到"竞品"/"竞争对手":
    → 模板 4
elif 用户提到"培训"/"通关"/"话术":
    → 模板 5
elif 文件是 .pdf 报告/白皮书/研报:
    → 模板 6
elif 输入是 URL:
    → 模板 7
elif 多文件 & 格式各异:
    → 模板 8
elif 用户提到"对比"/"vs":
    → 模板 11
else:
    → 模板 9（散装/通用，兜底）
```

## 场景模板

### 模板 1：PPT（无讲稿）
```
A 🟢 全拆（推荐）— 结构拆解 + 知识提取并行
B 🟡 仅捞知识 — 逐页提取关键信息入 vault
C 🔵 仅拆结构 — 叙事框架、页面类型、视觉风格
D ⚪ 自定义
```

### 模板 2：PPT（有讲稿 + 可识别演讲者）
```
A 🟢 三线全开（推荐）— 蒸馏 + 结构 + 知识，三条并行
B 🟡 蒸馏 + 知识 — 提炼人格 + 提取知识点
C 🔵 仅蒸馏 — 只看这个人怎么想、怎么讲
D ⚪ 自定义
```

### 模板 3：转录/演讲（单一明确人物）
```
A 🟢 蒸馏 + 知识并行（推荐）— 炼人格 + 捞知识
B 🟡 仅蒸馏 — 提炼心智模型/表达DNA
C 🔵 仅捞干货 — 知识点入 vault
D ⚪ 自定义
```

### 模板 4：竞品文档
```
A 🟢 全拆（推荐）— 知识提取 + 结构拆解并行
B 🟡 仅捞知识 — 竞品信息入 vault
C 🔵 仅拆结构 — 竞品的框架和手法
D ⚪ 自定义
```

### 模板 5：培训材料
```
A 🟢 学习→生成（推荐）— 先提取知识点入 vault，再用你的人格库生成话术
B 🟡 仅提取 — 知识点入 vault
C 🔵 逆向工程 — 分析为什么有效，提取可复用规律
D ⚪ 自定义
```

### 模板 6：纯文档/报告
```
A 🟢 深度提取（推荐）— 逐段提取入 vault，关联已有笔记
B 🟡 速览摘要 — 只存 3-5 条最关键结论
C 🔵 仅存档索引 — vault 只存文件路径 + 一句话描述
D ⚪ 自定义
```

### 模板 7：网页/链接
```
先静默 fetch → 按实际内容类型匹配模板 → 给选项
```

### 模板 8：混合批量
```
逐文件分类展示 → 选项：
A 🟢 全量并行（推荐）
B 🟡 分批处理
C 🔵 仅捞共同主题
D ⚪ 自定义
```

### 模板 9：散装笔记/想法
```
A 🟢 整理 + 提取（推荐）— 先结构化，再提取知识入 vault
B 🟡 仅整理 — 梳理结构，不改内容
C 🔵 直接生成 — 基于碎片生成完整内容
D ⚪ 自定义
```

### 模板 10：会议记录（多人）
```
A 🟢 提取决策+知识点（推荐）
B 🟡 逐人风格卡片 — 每人的表达风格分析
C 🔵 仅存档索引
D ⚪ 自定义
```

### 模板 11：对比分析
```
A 🟢 全拆+对比（推荐）— 各自拆解 + 并排对比差异
B 🟡 仅对比知识
C 🔵 仅对比结构
D ⚪ 自定义
```

## 蒸馏专属模板

### 情况一：用户直接说"蒸馏 XX"

```
Step 1 — 确认范围：
A 🟢 全面画像（推荐）— 六维全采，完整 SKILL.md + highlights
B 🟡 特定维度 — 表达力 / 决策力 / 内容创作
C 🔵 快速蒸馏 — 仅 highlights
D ⚪ 自定义

Step 2 — 确认用途（仅 A 和 B 需要）：
A 🟢 思维顾问  B 🟡 角色扮演  C 🔵 写作教练  D ⚪ 暂不确定

Step 3 — 确认素材范围：
A 🟢 全部纳入  B 🟡 仅本地  C 🔵 仅你刚给的  D ⚪ 我指定
```

### 情况二：用户提供内容资料 + 说"蒸馏"

```
A 🟢 蒸馏 + 知识并行（推荐）— 炼人格 + 捞知识入 vault
B 🟡 仅蒸馏 — 只看这个人怎么想、怎么表达
C 🔵 仅提取知识 — 捞干货入 vault，人格不关心
D ⚪ 自定义
```
