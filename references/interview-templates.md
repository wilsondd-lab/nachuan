# 纳川 · 通用访谈模板

## 设计原则

1. **先扫再问**：静默扫描内容，抓关键信号，再给倾向性建议。
2. **选择题，不开放题**：每个选项都是一个完整路径组合，用户只需选。
3. **推荐项在最前**：第一个选项是纳川判断最适合的路径。
4. **保留自定义**：每组模板最后一项都是自定义。
5. **不绑定行业**：模板只根据材料形态和用户目标路由。

---

## 信号扫描

| 扫描维度 | 内容 |
|----------|------|
| 文件类型 | PPT / PDF / Markdown / 文本 / 网页 / 表格 / 多文件 |
| 规模 | 页数、字数、文件数、时长 |
| 结构特征 | 是否有标题层级、目录、段落、页面、发言人、表格 |
| 关键词 | 项目名、主题词、任务词、对比词、生成要求 |
| 任务目标 | 看重点、拆结构、提知识、生成内容、做对比 |
| 知识库预搜 | 是否有可关联的既有笔记 |

---

## 模板匹配逻辑

```text
if 用户明确说“只看重点”:
    → 模板 1：精华萃取
elif 用户明确说“拆结构”:
    → 模板 2：结构拆解
elif 用户明确说“提取/入库/捞知识”:
    → 模板 3：知识提取
elif 用户明确说“写/生成/改成稿子”:
    → 模板 4：内容生成
elif 输入是 PPT / deck:
    → 模板 5：页面型材料
elif 输入是长文 / 报告 / PDF:
    → 模板 6：长文材料
elif 输入是转录 / 会议记录:
    → 模板 7：记录型材料
elif 输入是网页 / URL:
    → 先读取，再按实际内容匹配
elif 输入是多文件:
    → 模板 8：混合批量
elif 用户提到“对比 / vs / 差异”:
    → 模板 9：对比分析
else:
    → 模板 10：通用整理
```

## 场景模板

### 模板 1：精华萃取

```text
A 推荐：精华萃取 — 提炼核心观点、关键判断、可复用规律
B 备选：精华 + 知识 — 先看重点，再沉淀知识单元
C 更轻：速览摘要 — 只给 3-5 条最重要结论
D 自定义
```

### 模板 2：结构拆解

```text
A 推荐：完整拆解 — 叙事结构 + 表达模式 + 可复用框架
B 备选：只拆结构 — 看它怎么组织信息
C 更轻：只提模式 — 提取可复用表达套路
D 自定义
```

### 模板 3：知识提取

```text
A 推荐：深度提取 — 提取知识单元并建立关联
B 备选：精简提取 — 只保留关键知识点
C 更轻：索引入库 — 只存材料索引和一句话说明
D 自定义
```

### 模板 4：内容生成

```text
A 推荐：先提取再生成 — C → D，保证素材可追溯
B 备选：直接生成 — 基于当前材料产出初稿
C 更轻：生成提纲 — 只给结构和要点
D 自定义
```

### 模板 5：页面型材料

```text
A 推荐：全拆 — 结构拆解 + 知识提取并行
B 备选：只拆页面结构 — 看版式、节奏和叙事
C 更轻：只提知识 — 把页面里的关键信息沉淀出来
D 自定义
```

### 模板 6：长文材料

```text
A 推荐：精华 + 知识 — 提炼判断，并生成知识单元
B 备选：结构 + 知识 — 拆文章骨架并入库
C 更轻：速览摘要 — 只看核心结论
D 自定义
```

### 模板 7：记录型材料

```text
A 推荐：结论 + 决策 — 提炼共识、问题、行动项
B 备选：知识提取 — 把可复用内容写成知识单元
C 更轻：会议纪要 — 按主题整理
D 自定义
```

### 模板 8：混合批量

```text
A 推荐：逐文件分类处理 — 每份材料匹配最合适路径
B 备选：只提共同主题 — 找跨材料共性和差异
C 更轻：建立索引 — 先做文件目录和摘要
D 自定义
```

### 模板 9：对比分析

```text
A 推荐：全拆 + 对比 — 各自处理后并排比较
B 备选：只对比知识 — 聚焦事实、观点、结论差异
C 更轻：只对比结构 — 看表达方式和组织方式差异
D 自定义
```

### 模板 10：通用整理

```text
A 推荐：整理 + 萃取 — 先结构化，再提炼精华
B 备选：整理 + 入库 — 先结构化，再写知识单元
C 更轻：只整理 — 不扩写、不生成
D 自定义
```
