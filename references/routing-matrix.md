# 纳川 · 路由矩阵

## 路径定义

| 路径 | 名称 | 核心目标 |
|------|------|----------|
| A | 精华萃取 | 把材料中的重点、判断、规律提炼出来 |
| B | 结构拆解 | 还原材料的组织方式、叙事结构和表达模式 |
| C | 知识提取 | 把内容沉淀为 Markdown 知识单元 |
| D | 内容生成 | 基于已提取素材生成新的表达 |

---

## 完整路由表

| 输入类型 | 用户目标 | 推荐路径 | 执行模式 |
|----------|----------|----------|----------|
| PPT / deck | 全拆 | B + C | 并行 |
| PPT / deck | 看重点 | A | 单路径 |
| PPT / deck | 拆页面结构 | B | 单路径 |
| 长文 / 报告 | 提炼重点 | A + C | 并行 |
| 长文 / 报告 | 只入库 | C | 单路径 |
| 长文 / 报告 | 逆向表达 | B | 单路径 |
| 网页 / URL | 未明确 | fetch → 匹配模板 | 按类型 |
| 转录 / 会议记录 | 提炼结论 | A + C | 并行 |
| 转录 / 会议记录 | 纪要整理 | A | 单路径 |
| 表格 / 数据 | 提取洞察 | A + C | 并行 |
| 培训 / 说明材料 | 转成可用稿 | C → D | 串联 |
| 多文件混合 | 全量处理 | 逐文件分类 | 并行或分批 |
| 散装笔记 | 整理 | 整理 → A/C | 串联 |
| 对比分析 | 看差异 | B/C + 对比 | 并行 |
| 生成任务 | 直接出稿 | D | 单路径 |
| 生成任务 | 要可追溯 | C → D | 串联 |

---

## 路径组合模式

| 模式 | 何时使用 |
|------|----------|
| A | 只要重点、判断、规律 |
| B | 只看结构、框架、表达方式 |
| C | 只要可入库知识 |
| D | 只要生成结果 |
| A + C | 既要精华，也要知识沉淀 |
| B + C | 既拆结构，也提知识 |
| A + B + C | 高价值材料完整处理 |
| C → D | 需要先沉淀素材，再生成内容 |

---

## 通用领域处理

纳川不内置任何行业限定。遇到新领域时，只做三件事：

1. 从材料中提取领域词和标签。
2. 在知识库中查找已有相关笔记。
3. 按本次任务生成临时分类，必要时写入可复用标签建议。
