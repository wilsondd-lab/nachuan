# Path A · 人物蒸馏调度

## 调用目标

**Persona-Fusion**（继承女娲 Nuwa 方法论）：六维信息采集 → 心智模型三重验证 → SKILL.md + highlights + vault 精华笔记。

本路径需要 Persona-Fusion 技能包（独立安装）。

---

## 纳川层职责

纳川不执行蒸馏，只做三件事：
1. **识别**：判断输入是否触发蒸馏（用户说"蒸馏 XX" 或 内容包含明确人物 + 用户确认蒸馏）
2. **访谈**：用蒸馏专属模板确认范围/用途/素材
3. **调度**：将确认后的任务交给 Persona-Fusion 执行，必要时并行启动 Path C

---

## 两种触发方式

### 情况一：用户直接说"蒸馏 XX"

纳川走蒸馏专属访谈（3 步选择题：范围 → 用途 → 素材范围），确认后调度：

```
纳川 → Persona-Fusion
  Phase 1.0A（确认需求）
  → Phase 1.1（创建目录）
  → Phase 1.2（六维采集）
  → Phase 1.3（框架提炼 + 用户确认）
  → Phase 1.4（构建 SKILL.md）
  → Phase 1.5（精华写入 vault）
  → Phase 2（highlights 提取）
```

### 情况二：用户提供内容 + 说"蒸馏"

内容同时触发多条路径。纳川在选项中呈现"A 🟢 蒸馏 + 知识并行（推荐）"。

如果用户选了这条 → 纳川同时调度：
- **Sub-agent 1**：Path A — Persona-Fusion Phase 1
- **Sub-agent 2**：Path C — 知识提取协议

两条线共享同一份输入素材，并行执行。

## Vault 写入合约（Path A）

| 存什么 | vault 路径 | 大小 | 标签 |
|--------|-----------|------|------|
| 素材索引 | `人物精华/[人名]_素材索引.md` | ≤300字 | #纳川 #蒸馏 #素材索引 #[人名] |
| 提炼精华 | `人物精华/[人名]_精华.md` | ≤500字 | #纳川 #蒸馏 #精华 #[人名] |
| Skill 引用 | `人物精华/[人名]_Skill引用.md` | ≤300字 | #纳川 #蒸馏 #skill #[人名] |
| 可复用规律 | `人物精华/[人名]_规律.md` | ≤500字 | #纳川 #规律 #[人名] |

**绝不入 vault**：原始转录、完整 SKILL.md、六维调研文档（留在本地磁盘）。
