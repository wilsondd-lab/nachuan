# Path A · 精华萃取

## 定位

Path A 负责从任何输入中提炼最有复用价值的部分：核心观点、关键判断、方法步骤、反复出现的规律、值得保留的表达。

它不绑定特定对象、领域或下游系统。输入可以是文档、转录、网页、会议记录、方案、笔记或多文件材料包。

---

## 适用场景

| 输入 | 萃取重点 |
|------|----------|
| 长文 / 报告 | 核心论点、证据链、关键结论 |
| PPT / deck | 主线、判断、可复用表达 |
| 转录 / 会议记录 | 决策、共识、问题、后续动作 |
| 网页 / 文章 | 观点、结构、方法、案例 |
| 散装笔记 | 可命名的主题、规律、待办 |
| 多文件材料 | 跨材料共同主题和差异 |

---

## 输出结构

### 1. 精华摘要

用 300-800 字说明这份材料最值得带走的内容。

### 2. 核心判断

列出 3-7 条判断，每条包含：

- 判断本身
- 支撑依据
- 适用边界

### 3. 可复用规律

把材料中能迁移到其他任务的规律命名并解释。

### 4. 后续用途

说明这些精华可以继续用于知识提取、结构拆解或内容生成。

---

## 与其他路径的关系

| 组合 | 用法 |
|------|------|
| A + C | 同时获得精华摘要和可入库知识单元 |
| A + B | 同时获得核心观点和结构方法 |
| A → D | 先萃取重点，再生成面向具体场景的表达 |
| A + B + C | 对高价值材料做完整处理 |

---

## 写入合约

| 存什么 | 知识库路径 | 大小 | 标签 |
|--------|------------|------|------|
| 精华摘要 | `精华萃取/[材料名]_精华.md` | 300-800字 | `纳川`, `Path-A`, `精华萃取` |
| 核心判断 | `精华萃取/[材料名]_判断.md` | 3-7条 | `纳川`, `判断`, `Path-A` |
| 可复用规律 | `精华萃取/[规律名].md` | 300-500字 | `纳川`, `规律`, `Path-A` |

完整中间分析、原始材料和临时草稿不进入知识库。
