# Path D · 内容生成

## 定位

Path D 基于已经提取的知识、结构和精华，生成新的表达：文案、脚本、说明、提纲、模板、草稿。

它不依赖预设风格或行业模板。用户可以指定语气、受众、长度、渠道和格式。

---

## 适用场景

| 场景 | 输入 | 产出 |
|------|------|------|
| 文案生成 | 知识单元 + 输出目标 | 标题、短文、长文 |
| 脚本生成 | 精华摘要 + 结构框架 | 视频脚本、讲稿、旁白 |
| 说明生成 | 材料重点 + 受众信息 | FAQ、指南、说明文 |
| 模板生成 | 可复用结构 | SOP、表格、检查清单 |
| 改写润色 | 原稿 + 风格约束 | 更清晰的版本 |

---

## 生成前确认

最多补问一题，优先用选择题：

```text
A 面向新读者，讲清楚
B 面向内部协作，讲准确
C 面向公开传播，讲得有吸引力
D 自定义
```

---

## 调度方式

| 模式 | 用法 |
|------|------|
| D | 用户已有明确素材和输出目标 |
| C → D | 需要先提取知识，再生成内容 |
| A → D | 需要先抓重点，再生成表达 |
| B → D | 需要先复用结构，再生成稿件 |

---

## 写入合约

| 存什么 | 知识库路径 | 大小 | 标签 |
|--------|------------|------|------|
| 生成品索引 | `生成内容/[标题]_索引.md` | 200-400字 | `纳川`, `Path-D`, `内容生成` |
| 可复用模板 | `生成内容/[模板名].md` | 300-800字 | `纳川`, `模板`, `Path-D` |

生成全文、交付稿和临时版本默认留在本地磁盘。
