# Path D · 内容生成调度

## 定位

基于 Obsidian vault 中已沉淀的知识 + 已蒸馏的人格，生成一线可用的内容。

**前置依赖**：Path A（有人格）+ Path C（有知识素材）。如果没有，需先走 A 或 C。

---

## 适用场景

| 场景 | 典型输入 | 产出 |
|------|----------|------|
| 产品话术生成 | vault 中的产品知识点 + 蒸馏人格 | 一线销售话术 |
| 培训内容转化 | vault 中的培训知识点 + 专家人格 | 通俗化培训脚本 |
| 短视频脚本 | vault 中的核心卖点 + 大众传播人格 | 3 分钟即席演讲稿 |
| 长文/报告 | vault 中的多源知识 + 深度分析人格 | 结构化长文 |

---

## 调度方式

### 独立调度（用户明确要生成内容）

纳川确认：
- 生成什么类型？
- 用什么人格？（从 vault `人物精华/` 中列出可用人格）
- 知识来源？（从 vault `知识记忆/` 指定主题）
- 长度/格式要求？

→ 从 vault 加载知识笔记 + 人格精华 → 传递给生成管线 → 生成初稿 → 打磨 → 交付

### 串联调度（培训材料，Path C → D）

模板 5 的 A 选项：
1. Path C 先执行：提取培训材料知识点 → vault
2. Path D 后执行：从 vault 加载知识点 + 用户选择人格 → 生成

---

## 生成前检查清单

- [ ] vault 中有已蒸馏人格？（查 `人物精华/`）
- [ ] vault 中有相关知识笔记？（查 `知识记忆/`）
- [ ] 用户指定了输出格式和长度？

---

## Vault 写入合约（Path D）

| 存什么 | vault 路径 | 大小 | 标签 |
|--------|-----------|------|------|
| 生成品索引 | `生成内容/[标题]_索引.md` | ≤300字 | #纳川 #内容生成 #[类型] |
| 可复用话术模板 | `生成内容/[模板名].md` | ≤500字 | #纳川 #话术模板 #[场景] |

生成的全文**不入 vault**，留本地磁盘。
