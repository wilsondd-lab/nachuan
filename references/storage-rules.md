# 纳川 · 存储规则（Obsidian 版）

## 三条存储轨道

| 轨道 | 定位 | 原则 |
|------|------|------|
| **Obsidian Vault** | 知识网络 | 决策、知识点、框架摘要、人物精华；可被未来检索和复用；wikilinks 关联 |
| **项目记忆** | 会话间状态 | 路由决策、偏好、进度；可存入 vault 的 `纳川/纳川_MOC.md` 或独立笔记 |
| **本地磁盘** | 完整内容存储 | 分析文档、原始素材、生成成品；不入 vault，不占笔记图谱 |

---

## 判定流程

```
产出物
  │
  ├─ 是可被未来检索的知识/框架/决策？
  │   └─ 是 → Obsidian Vault（wikilinks + tags）
  │
  ├─ 是项目状态/进度？
  │   └─ 是 → 项目记忆（vault 独立笔记）
  │
  ├─ 是完整分析文档/原始素材/成品？
  │   └─ 是 → 本地磁盘
  │
  └─ 不确定？
      └─ 默认本地磁盘，纳川在写入确认时提醒用户决定
```

## 各路径默认存储

| 路径 | Vault 目录 | 本地目录（不入 vault） |
|------|-----------|----------------------|
| A · 蒸馏 | `vault/纳川/人物精华/` | `~/.hermes/skills/creative/persona-fusion/personas/[name]/` |
| B · 结构拆解 | `vault/纳川/结构拆解/` | `~/Documents/Claude ppt engine/02_拆解沉淀/[项目名]/` |
| C · 知识提取 | `vault/纳川/知识记忆/` + `vault/纳川/学习报告/` | `~/Documents/纳川/学习报告/` |
| D · 内容生成 | `vault/纳川/生成内容/`（仅索引） | 用户指定，默认 `~/Documents/纳川/生成内容/` |

## Vault 目录结构

```
your-vault/
└── 纳川/
    ├── 纳川_MOC.md              # 总索引（Map of Content）
    ├── 学习报告/                 # Path C 产出
    │   └── 文档名_日期.md
    ├── 知识记忆/                 # Path C 产出
    │   ├── 保险/
    │   ├── 养老/
    │   └── ...
    ├── 人物精华/                 # Path A 产出
    ├── 结构拆解/                 # Path B 产出
    ├── 生成内容/                 # Path D 索引
    └── 路由日志/                 # 项目记忆
```

---

## 不存什么

以下内容**不存储**（处理完即丢弃）：
- 中间过程的临时笔记（除非用户要求保留）
- 重复的知识笔记（Phase 3 去重判断为"重复"的）
- 失败/中断的执行记录
- 纯技术性的工具调用日志
