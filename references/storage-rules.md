# 纳川 · 存储规则

## 三条存储轨道

| 轨道 | 定位 | 原则 |
|------|------|------|
| **知识库** | 长期可复用资产 | 精华、知识单元、结构摘要、模板、决策 |
| **路由日志** | 任务过程记录 | 输入、扫描结论、用户选择、执行结果 |
| **本地磁盘** | 完整材料和交付稿 | 原始文件、完整分析、生成全文、临时草稿 |

---

## 判定流程

```text
产出物
  ├─ 未来会被检索或复用？
  │   └─ 是 → 写入知识库
  ├─ 记录任务过程和选择？
  │   └─ 是 → 写入路由日志
  ├─ 是完整交付稿、原始材料或中间草稿？
  │   └─ 是 → 留在本地磁盘
  └─ 不确定？
      └─ 默认留本地，写入前请用户确认
```

---

## 各路径默认存储

| 路径 | 知识库目录 | 本地目录 |
|------|------------|----------|
| A · 精华萃取 | `纳川/精华萃取/` | 用户指定或任务目录 |
| B · 结构拆解 | `纳川/结构拆解/` | 用户指定或任务目录 |
| C · 知识提取 | `纳川/学习报告/` + `纳川/知识单元/` | 用户指定或任务目录 |
| D · 内容生成 | `纳川/生成内容/`（索引和模板） | 用户指定或任务目录 |

---

## 推荐目录结构

```text
your-vault/
└── 纳川/
    ├── 纳川_MOC.md
    ├── 学习报告/
    ├── 知识单元/
    ├── 精华萃取/
    ├── 结构拆解/
    ├── 生成内容/
    └── 路由日志/
```

---

## 不默认写入

- 原始材料
- 完整生成稿
- 临时草稿
- 工具调用日志
- 重复知识点
- 未经用户确认的知识库写入
