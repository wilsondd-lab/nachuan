# 纳川.skill

> To be anyone, to be better you.

[中文](README.md) · [English](README.en.md) · [日本語](README.ja.md) · [한국어](README.ko.md) · [Français](README.fr.md) · [Deutsch](README.de.md) · [Português](README.pt.md) · [العربية](README.ar.md)

**Distill Everything.skill** es el nombre en inglés de **纳川.skill**.

纳川.skill es una Skill universal de Claude Code para destilar materiales. Lee PDFs, presentaciones, páginas web, transcripciones, paquetes de investigación, borradores y notas dispersas, y los convierte en conocimiento, estructura y activos de contenido reutilizables.

No es una colección de prompts. Es un flujo de trabajo: primero escanea, luego enruta, ejecuta la tubería adecuada y conserva el resultado para reutilizarlo.

---

## Por qué se llama Nachuan

Nachuan evoca la idea de recibir muchas corrientes.

El trabajo real rara vez llega como entrada limpia. Llega mezclado: documentos, gráficos, enlaces, transcripciones, ideas incompletas y fragmentos. 纳川.skill primero entiende qué hay dentro y después decide qué ruta hará que el material sea útil.

**To be anyone, to be better you** no significa copiar a alguien. Significa absorber métodos, estructuras, criterios y formas de expresión de muchas fuentes para fortalecer tu propio trabajo.

---

## Tres puntos fuertes

| Punto fuerte | Problema que resuelve | Experiencia |
|---|---|---|
| **Escanea antes de preguntar** | Con materiales complejos, el usuario quizá no sabe qué pedir | Ofrece una vista general y 2-4 opciones concretas |
| **Cuatro tuberías con ruta clara** | Resumen, estructura, conocimiento y generación suelen mezclarse | Cada ruta tiene objetivo, salida y siguiente paso |
| **El resultado se vuelve activo** | Muchas respuestas de IA se pierden en una conversación | Guarda Markdown, etiquetas, enlaces y plantillas reutilizables |

---

## Cuatro tuberías

| Path | Nombre | Cuándo usarla | Salida típica |
|---|---|---|---|
| **A** | Essence Distillation | Necesitas tesis, criterios y principios reutilizables | Resumen, lista de decisiones, tarjetas de principios |
| **B** | Structure Decomposition | Quieres entender narrativa, página o argumento | Mapa estructural, patrón de expresión, marco reutilizable |
| **C** | Knowledge Extraction | Quieres convertir material en base de conocimiento | Notas Markdown, etiquetas YAML, `[[wikilinks]]` |
| **D** | Content Generation | Quieres crear nueva expresión desde conocimiento extraído | Copy, guiones, esquemas, explicaciones, plantillas, borradores |

---

## Flujo de trabajo

1. **Añade material**: documentos, enlaces, transcripciones, notas o paquetes mixtos.
2. **Escaneo silencioso**: detecta tipo, escala, estructura, palabras clave y valor posible.
3. **Elige una ruta**: recibe una recomendación y alternativas.
4. **Ejecuta la tubería**: usa A/B/C/D por separado, en paralelo o en secuencia.
5. **Conserva el resultado**: guárdalo como Markdown buscable, enlazable y reutilizable.

## Origen

纳川.skill fue inspirado por Nuwa Skill de Huashu. La idea clave: una Skill puede ser más que un prompt. Puede convertir experiencia, criterio, selección de rutas y flujo de trabajo en un sistema reutilizable.

Sobre esa inspiración, 纳川.skill pasó por cuatro o cinco iteraciones prácticas y se acercó más a las necesidades reales del trabajo de primera línea.
