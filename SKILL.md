# 纳川 · Distill Everything

**纳川** 是内容处理体系的入口层。任何材料进入后，先扫描信号，再给出明确选项，最后调度到通用管线执行。目标不是一次性总结，而是把输入沉淀为可复用的知识、结构和内容资产。

---

## 一、四条通用管线

| 路径 | 名称 | 做什么 | 典型产出 |
|------|------|--------|----------|
| **A** | 精华萃取 | 提炼核心观点、关键判断、可复用规律 | 精华摘要、规律卡片、判断列表 |
| **B** | 结构拆解 | 拆解叙事、框架、页面或段落组织方式 | 结构图、表达模式、框架摘要 |
| **C** | 知识提取 | 把事实、机制、方法提取为 Markdown 知识单元 | 学习报告、知识笔记、wikilinks |
| **D** | 内容生成 | 基于已沉淀素材生成新的表达 | 文案、脚本、说明、模板、草稿 |

---

## 二、前置要求

纳川默认使用 Obsidian vault 或任意 Markdown 文件夹作为知识库后端。推荐结构：

```text
your-vault/
└── 纳川/
    ├── 学习报告/
    ├── 知识单元/
    ├── 精华萃取/
    ├── 结构拆解/
    ├── 生成内容/
    ├── 路由日志/
    └── 纳川_MOC.md
```

## 三、快速路由

### 按内容类型

| 输入 | 默认推荐 | 说明 |
|------|----------|------|
| PPT / deck | B + C 并行 | 结构拆解 + 知识提取 |
| 长文 / 报告 | A + C 并行 | 提炼核心判断 + 提取知识单元 |
| 网页 / 链接 | fetch 后匹配 | 先读取，再按实际内容路由 |
| 转录 / 会议记录 | A + C 并行 | 提炼结论 + 提取决策与知识 |
| 培训 / 说明材料 | C → D 串联 | 先沉淀知识，再生成可用表达 |
| 多文件混合 | 逐文件分类 | 分批或并行处理 |
| 散装笔记 | 整理 → A/C | 先结构化，再萃取或提取 |
| 对比分析 | B/C + 对比 | 各自处理后并排比较 |

### 按用户指令

| 指令 | 行为 |
|------|------|
| “只看重点” | 推荐 A |
| “拆结构” | 推荐 B |
| “捞知识 / 入库” | 推荐 C |
| “写成内容 / 生成稿子” | 推荐 D 或 C → D |
| “全拆 / 都要” | 推荐 A + B + C |

## 四、访谈流程

```text
Step 0: 静默扫描
  - 识别文件类型、规模、结构、关键词
  - 搜索已有知识，准备关联
  - 匹配场景模板

Step 1: 扫描结论 + 选择题
  A 推荐路径
  B 备选路径
  C 更轻路径
  D 自定义

Step 2: 必要时补一题
  - 输出用途 / 知识库范围 / 生成格式

Step 3: 执行完成后确认沉淀
  - 展示预览
  - 用户选择：全部写入 / 仅写知识 / 仅写结构 / 暂不写入
```

**设计原则**：
- 先扫再问
- 只问选择题
- 第一个选项永远是推荐项
- 每个选项都对应完整路径组合

---

## 五、执行模式

| 模式 | 用法 |
|------|------|
| 单路径 | 用户目标明确，例如只提取知识 |
| 双并行 | 结构和知识同时处理 |
| 三并行 | 精华、结构、知识同时处理 |
| 串联 | 后一步依赖前一步结果，例如 C → D |

---

## 六、写入合约

| 路径 | 存入知识库 | 留在本地磁盘 |
|------|------------|--------------|
| A · 精华萃取 | 精华摘要、规律卡片、判断索引 | 完整中间分析 |
| B · 结构拆解 | 结构摘要、模式索引、可复用框架 | 完整拆解文档、截图素材 |
| C · 知识提取 | 学习报告、知识单元、wikilinks | 逐页草稿、原始素材 |
| D · 内容生成 | 生成品索引、可复用模板 | 生成全文、交付稿 |

写入前必须展示预览，并让用户选择确认。
