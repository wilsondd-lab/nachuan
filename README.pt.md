# 纳川.skill

> To be anyone, to be better you.

[中文](README.md) · [English](README.en.md) · [日本語](README.ja.md) · [한국어](README.ko.md) · [Español](README.es.md) · [Français](README.fr.md) · [Deutsch](README.de.md) · [العربية](README.ar.md)

**Distill Everything.skill** é o nome em inglês de **纳川.skill**.

纳川.skill é uma Skill universal para Claude Code que destila materiais. Ela lê PDFs, apresentações, páginas web, transcrições, pacotes de pesquisa, rascunhos e notas soltas, transformando tudo em conhecimento, estrutura e ativos de conteúdo reutilizáveis.

Não é uma pilha de prompts. É um fluxo de trabalho: primeiro escanear, depois rotear, executar a pipeline certa e guardar o resultado para reuso.

---

## Por que Nachuan

Nachuan traz a ideia de receber muitos fluxos.

O trabalho real raramente chega como uma entrada limpa. Ele vem em documentos, gráficos, links, transcrições, ideias pela metade e fragmentos. 纳川.skill entende primeiro o que existe ali e depois decide qual rota torna o material útil.

**To be anyone, to be better you** não significa copiar outra pessoa. Significa absorver métodos, estruturas, critérios e formas de expressão de várias fontes para fortalecer o seu próprio trabalho.

## Três destaques

| Destaque | Problema resolvido | Experiência |
|---|---|---|
| **Escaneia antes de perguntar** | Com material confuso, o usuário talvez não saiba o que pedir | Mostra uma visão geral e 2-4 opções concretas |
| **Quatro pipelines roteadas** | Resumo, estrutura, conhecimento e geração costumam se misturar | Cada rota tem objetivo, saída e próximo passo claros |
| **Saída vira ativo** | Muitas respostas de IA desaparecem em uma conversa | Markdown, tags, links e modelos ficam reutilizáveis |

---

## Quatro pipelines

| Path | Nome | Quando usar | Saída típica |
|---|---|---|---|
| **A** | Essence Distillation | Para capturar ideias centrais, critérios e princípios reutilizáveis | Resumo, lista de decisões, cartões de princípios |
| **B** | Structure Decomposition | Para entender narrativa, página, argumento ou organização | Mapa estrutural, padrão de expressão, framework |
| **C** | Knowledge Extraction | Para criar uma base de conhecimento duradoura | Notas Markdown, tags YAML, `[[wikilinks]]` |
| **D** | Content Generation | Para criar nova expressão a partir do conhecimento extraído | Texto, roteiros, outlines, explicações, modelos, rascunhos |

---

## Fluxo de trabalho

1. **Adicionar material**: documentos, links, transcrições, notas ou pacotes mistos.
2. **Escaneamento silencioso**: detectar tipo, escala, estrutura, palavras-chave e valor possível.
3. **Escolher rota**: receber uma recomendação e alternativas.
4. **Executar pipeline**: usar A/B/C/D sozinho, em paralelo ou em sequência.
5. **Guardar resultado**: salvar como Markdown pesquisável, linkável e reutilizável.

## Origem

纳川.skill foi inspirado pela Nuwa Skill de Huashu. A ideia principal: uma Skill pode ser mais do que um prompt. Ela pode transformar experiência, julgamento, escolha de rotas e workflow em um sistema reutilizável.

A partir dessa inspiração, 纳川.skill passou por quatro ou cinco iterações práticas e ficou mais próximo das necessidades reais do trabalho de linha de frente.
