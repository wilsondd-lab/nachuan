# 纳川.skill

> To be anyone, to be better you.

[中文](README.md) · [English](README.en.md) · [日本語](README.ja.md) · [한국어](README.ko.md) · [Español](README.es.md) · [Français](README.fr.md) · [Deutsch](README.de.md) · [Português](README.pt.md)

**Distill Everything.skill** هو الاسم الإنجليزي لـ **纳川.skill**.

纳川.skill هي مهارة عامة لـ Claude Code لتقطير المواد. تقرأ ملفات PDF والعروض التقديمية وصفحات الويب والنصوص المفرغة وحزم البحث والمسودات والملاحظات المتفرقة، ثم تحولها إلى معرفة وبنية وأصول محتوى قابلة لإعادة الاستخدام.

ليست مجموعة من المطالبات. إنها سير عمل: قراءة أولا، ثم اختيار المسار، ثم تنفيذ pipeline مناسبة، ثم حفظ النتيجة بشكل قابل لإعادة الاستخدام.

---

## معنى الاسم

Nachuan يحمل معنى استقبال تيارات كثيرة.

في العمل الحقيقي لا تصل المواد بشكل مرتب دائما. قد تأتي كمستندات ورسوم وروابط ونصوص مفرغة وأفكار غير مكتملة وشظايا ملاحظات. يبدأ 纳川.skill بفهم ما يوجد داخل المواد، ثم يقرر أي مسار يجعلها مفيدة.

**To be anyone, to be better you** لا تعني نسخ شخص آخر. معناها استيعاب الأساليب والبنى والأحكام وطرق التعبير من مصادر مختلفة لتقوية عملك أنت.

## ثلاث نقاط قوة

| القوة | المشكلة التي تحلها | التجربة |
|---|---|---|
| **يفحص قبل أن يسأل** | عندما تكون المواد فوضوية قد لا يعرف المستخدم ماذا يطلب | يقدم نظرة عامة ثم 2-4 خيارات معالجة واضحة |
| **أربع pipelines موجهة** | التلخيص والبنية واستخراج المعرفة والتوليد تختلط كثيرا | لكل مسار هدف ومخرج وخطوة تالية |
| **المخرجات تصبح أصولا** | كثير من مخرجات الذكاء الاصطناعي تختفي داخل محادثة واحدة | يحفظ Markdown ووسوما وروابط وقوالب قابلة لإعادة الاستخدام |

---

## أربع pipelines

| Path | الاسم | متى تستخدمها | المخرج المعتاد |
|---|---|---|---|
| **A** | Essence Distillation | عند الحاجة إلى الأفكار الأساسية والأحكام والمبادئ القابلة لإعادة الاستخدام | ملخص، قائمة قرارات، بطاقات مبادئ |
| **B** | Structure Decomposition | عند الحاجة إلى فهم السرد أو الصفحة أو الحجة | خريطة بنية، نمط تعبير، إطار قابل لإعادة الاستخدام |
| **C** | Knowledge Extraction | عند تحويل المواد إلى قاعدة معرفة طويلة الأمد | ملاحظات Markdown، وسوم YAML، `[[wikilinks]]` |
| **D** | Content Generation | عند إنشاء تعبير جديد من معرفة مستخرجة | نصوص، سكربتات، مخططات، شروحات، قوالب، مسودات |

---

## سير العمل

1. **إضافة المواد**: مستندات أو روابط أو نصوص مفرغة أو ملاحظات أو حزمة مختلطة.
2. **فحص صامت**: تحديد النوع والحجم والبنية والكلمات المفتاحية والقيمة المحتملة.
3. **اختيار المسار**: توصية واضحة مع بدائل.
4. **تنفيذ pipeline**: تشغيل A/B/C/D بشكل منفرد أو متواز أو متسلسل.
5. **حفظ النتيجة**: تخزينها كملفات Markdown قابلة للبحث والربط وإعادة الاستخدام.

## الأصل

استلهم 纳川.skill من Nuwa Skill الخاصة بـ Huashu. الفكرة الجوهرية: يمكن أن تكون Skill أكثر من مجرد prompt. يمكنها تحويل الخبرة والحكم واختيار المسار وسير العمل إلى نظام قابل لإعادة الاستخدام.

بناء على هذا الإلهام، مر 纳川.skill بأربع أو خمس تكرارات عملية حتى صار أقرب إلى احتياجات العمل الحقيقي مع المواد المعقدة.
